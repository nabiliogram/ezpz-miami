Data collection
Dedicated scrapers visit Miami event sources — museums, parks, libraries, ticket platforms — and pull their current event listings into a normalized format.

Relevance filtering
Not all scraped events belong on the site. Each event is checked against audience, geography, date range, and content rules. Only kid-friendly, upcoming, Miami-Dade events pass through.
2
Phase 2 Event-Level Filtering deterministic
Every scraped event passes through validation gates in helpers.py. Events that fail any check are dropped before touching the database.
Filter Gates
  • ✓ Required fields — must have title + start_date
  • ✓ Time window — 7-day lookback to 14-day lookahead
  • ✓ Kid-friendly — 65+ adult keyword blocklist (nightclub, cocktail, 21+, networking...)
  • ✓ Commitment filter — rejects multi-week/semester programs
  • ✓ Late-night filter — drops events starting ≥ 6 PM
  • ✓ Geo-bound — lat/lng must fall in Miami-Dade bounding box
  • ✗ Age gate — extracted age_min ≥ 13 → hard exclude
Outcome
PASS → upsert to Supabase
FAIL → skipped, logged

Cross-source deduplication
The same event often appears on multiple sources. The system groups matches across sources and retains the highest-quality record based on a scoring model.
3
Phase 3 Cross-Source Deduplication ~5 sec
Groups events by normalized title + datetime + address. Scores each record on data quality and keeps the winner. Runs before enrichment to avoid wasting LLM tokens on duplicates.
Grouping Key
normalize(title) + date_to_minute + normalize(address)

Fuzzy: 70% word overlap (stop words excluded) within same (datetime, address).

AI editorial enrichment
Raw event data is rewritten by an LLM into parent-facing editorial content: a headline, a summary focused on the child's experience, practical logistics, and structured metadata like activity type and neighborhood.
4
Phase 4 LLM Enrichment — Claude Haiku ~0.3s / event
All events where enriched_at IS NULL are sent one-by-one to Claude Haiku with a detailed brand voice prompt. This is the editorial engine.
System Prompt
"You are the editorial voice of EZPZ Miami. Warm, witty, direct. Like a trusted friend who's already vetted the event — honest, specific, no marketing fluff. Think NYT Parenting meets a local Miami parent newsletter."
User Prompt (per event)
title + venue + date/time + current age range + source name + up to 3,000 chars of description. Eventbrite events get full page fetched first.
Raw Input
title: "Spring Break Art Camp for Kids"
venue: "Perez Art Museum Miami"
description: "Join us for a week of creative exploration... registration required... ages 5-10... $45/day..."
age_min: 0
age_max: null
→
LLM Output (JSON)
editorial_headline: "Little Artists Take Over PAMM This Spring Break"
parent_summary: "Kids 5-10 get messy with paint, collage, and sculpture in PAMM's studio. Drop-off friendly, $45/day includes materials."
practical_notes: "Registration required. Parking in museum garage. Pack a snack."

age_min/max: 5 / 10 (confidence: high)
Post-Enrichment Safety
Scans for private group signals (members-only, approval required) → deletes event
Validates neighborhood against known Miami area names → nulls if invalid
Strips phone numbers & emails from editorial text before DB write
Age refinement logic: LLM ages only overwrite DB when confidence is high/medium AND current values are generic defaults (0/null). Scraper-provided specific ages are never overwritten. Month-based values (e.g., "6 months" → age_min: 6) are caught and corrected in post-processing.
enriched events → deterministic validation pass
Quality control
A set of deterministic checks runs against all enriched events to catch edge cases: adult content that slipped through, broken links, invalid age ranges, and missing metadata.
5
Phase 5 Post-Enrichment Validation deterministic, no LLM
Regex, numeric range, and HTTP status checks against all enriched events. No LLM calls. Catches anything the enricher missed or introduced.

Expiry and cleanup
Past events are flagged and removed from the live feed. They remain in the database for historical tracking but are excluded from all user-facing queries.
6
Phase 6 Cleanup & Expiry bulk UPDATE
All events where start_date < today are bulk-updated to is_past = true. Remain in DB for historical reference, excluded from frontend queries. Frontend filter: is_canceled=false AND is_kid_friendly=true AND start_date ≥ today.
database ready → frontend reads live
